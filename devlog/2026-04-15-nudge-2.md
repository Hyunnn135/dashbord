## 📌 오늘 한 일

Phase 0(기획)을 마치고 Xcode 프로젝트까지 만들어둔 상태에서, 오늘 드디어 템플릿 걷어내고 Nudge 컨셉 자체를 붙였다.

**1) 디스크 풀 & 빌드 실패 해결**

오전에 시뮬레이터 빌드가 `No space left on device` 로 죽었다. Xcode `DerivedData` + `ModuleCache` 가 수십 GB 먹고 있어서 아래 두 줄로 정리:

```bash
rm -rf ~/Library/Developer/Xcode/DerivedData
rm -rf ~/Library/Caches/com.apple.dt.Xcode
```

이후 Clean Build Folder → 다시 빌드해서 기본 템플릿 (Apple 기본 위젯 "Time + Favorite Emoji" + SwiftData Items 리스트) 은 정상 동작 확인.

**2) Watch 타겟 임시 분리**

Phase 3(워치) 들어가기 전까지는 iOS 빌드가 Watch 앱을 함께 굽지 않도록 분리했다. 타겟 자체는 프로젝트에 그대로 있고, `nudge` iOS 타겟의 `Embed Watch Content` 빌드 페이즈 / `NudgeWatch` 타겟 의존성만 `project.pbxproj` 에서 제거. Phase 3 재개 시 두 줄 되돌리면 끝. 백업 파일(`project.pbxproj.backup-pre-watch-disable`) 남겨둠.

**3) App Group 양쪽 설정**

위젯 쪽 `NudgeWidget/NudgeWidget.entitlements` 새로 만들고 `group.site.salarykorea.nudge` 추가. pbxproj 의 NudgeWidgetExtension Debug/Release 빌드 설정에 `CODE_SIGN_ENTITLEMENTS = NudgeWidget/NudgeWidget.entitlements;` 항목 주입. iOS 앱 쪽은 이미 같은 App Group 으로 설정되어 있어서 그대로 둠.

**4) Exercise + SharedStore 공용 코드 (핵심)**

`Exercise { pushup, pullup, squat }` enum과 App Group UserDefaults 기반 `SharedStore` 를 작성. 이 둘이 앱과 위젯 양쪽에서 같은 데이터를 읽고 쓴다.

```swift
enum SharedStore {
    static let appGroupID = "group.site.salarykorea.nudge"
    static var activeExercise: Exercise { ... }
    static func count(for: Exercise, on: Date) -> Int { ... }
    @discardableResult
    static func increment(_: Exercise) -> Int { ... }
    @discardableResult
    static func decrement(_: Exercise) -> Int { ... }
}
```

저장 포맷은 `[ "YYYY-MM-DD": [exerciseRaw: count] ]` 단일 JSON. Phase 4 통계 들어갈 때 쉽게 SwiftData/JSON 파일로 마이그레이션 가능한 구조로 둠.

**5) 메인 앱 ContentView 재작성**

기존 SwiftData Items 리스트 템플릿을 전부 걷어내고 Nudge 메인 화면:

- 상단: 헤더 + 날짜 + 세그먼트 Picker(푸시업/풀업/스쿼트)
- 중앙: 큰 숫자(오늘 횟수) + 운동 이름
- 하단: 거대한 +1 버튼 + −1 취소 버튼

햅틱은 +1 에 `.medium`, −1 에 `.light`. 앱이 foreground 로 돌아올 때(`didBecomeActiveNotification`) 와 운동 변경 시 `SharedStore` 에서 다시 읽어오도록 해서 위젯에서 찍힌 값이 바로 반영되게 함.

**6) 위젯 재작성 — 템플릿 → Interactive 3버튼**

- 소형 위젯: 활성 운동 1개만 크게. 위젯 전체 탭 = 해당 운동 +1
- 중형 위젯: 3개 운동 카드 병렬. 각 카드 탭 = 해당 운동 +1
- `AppIntentConfiguration` → `StaticConfiguration` 으로 전환 (사용자 구성 옵션 없음)
- 타임라인 폴리시: `.after(nextMidnight)` — 자정 되면 자동 리셋되어 새 엔트리 생성
- 앱에서 +1 할 때마다 `WidgetCenter.shared.reloadAllTimelines()` 호출

## 🤔 결정 사항

- **공용 코드는 파일 복제**: `Exercise` + `SharedStore` 를 `nudge/` 와 `NudgeWidget/` 양쪽에 동일 내용으로 두 개 유지. `Shared/` 폴더를 만들고 `fileSystemSynchronizedGroup` 을 두 타겟에 연결하는 게 더 깔끔하지만, pbxproj 구조 편집 리스크 대비 이득이 작음. 파일 상단에 "한쪽 수정 시 다른 쪽도 같이 수정" 주석. 파일 수가 늘면 그때 가서 Swift Package 로 빼기.
- **Phase 1 데이터 저장소 = App Group UserDefaults**: SwiftData + App Group 조합이 가능하긴 하지만 위젯 extension 에서 알려진 자잘한 이슈가 있고, MVP 기능엔 UserDefaults 로 충분. Phase 4(통계)에서 히스토리가 필요해지면 그때 SwiftData 또는 JSON 파일로 이전.
- **위젯 Configuration = Static**: 템플릿의 `AppIntentConfiguration` 은 "favorite emoji" 같은 사용자 구성용이라 Nudge 에는 불필요. 꾹 눌러 편집 → 활성 운동 변경은 별도 `SetActiveExerciseIntent` 로 준비해뒀지만 아직 UI 연결 안 함 (Phase 2 후반에).

## 💭 느낀 점

Phase 0 기획에서 "위젯 탭 1회 = 운동 1회 기록" 라는 한 문장을 가장 먼저 고정해두길 잘했다. 실제 구현하면서 "위젯 꾹 눌러 운동 변경" 같은 부차 기능에 대한 스펙 논쟁이 0이었음. 기획이 단단하니 코드가 빠르게 떨어진다.

위젯 Interactive 는 생각보다 조립이 간단했다. `Button(intent: SomeAppIntent(...))` 한 줄로 탭 처리 끝. iOS 17 이전엔 위젯 탭 = 앱 deeplink 였는데 App Intents 이후로 진짜 위젯 안에서 로직이 돈다. 본 앱이 잠들어 있어도.
